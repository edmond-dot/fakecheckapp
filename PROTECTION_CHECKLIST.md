Before you click "Make this repository public" on GitHub, do these in order. Total time: ~30 minutes.

## ✅ TODAY — Before Publishing (30 min total)

### ☑️ Step 1: Email yourself the ZIP (1 min) — DO THIS FIRST
- Send `fakecheckapp_github.zip` to your own Gmail address
- Subject line: `FakeCheckApp v1 — Original Work — Edmond Azbel — [TODAY'S DATE]`
- Body: `This is my original work, created by Edmond Azbel on [DATE]. All rights reserved.`
- **Why:** Gmail timestamps are court-admissible evidence of authorship date in Canada. Before publishing, this creates your first immutable record.

### ☑️ Step 2: Save a local copy with date in filename (1 min)
- On your computer, rename the folder to: `fakecheckapp_v1_2026-06-01_edmond_azbel`
- Make 2-3 copies (USB drive, cloud storage like Google Drive/Dropbox)
- **Why:** Multiple dated copies = stronger evidence

### ☑️ Step 3: Create GitHub account (3 min)
- Go to https://github.com/signup
- Use a username that reflects you: `edmondazbel` or `eazbel`
- Use a strong password — your repo is your intellectual property
- Enable 2-Factor Authentication (Settings → Password and authentication → 2FA)
- **Why:** Account security = your name stays attached to commits

### ☑️ Step 4: Create repo as PRIVATE first (2 min)
- Click "+" → "New repository"
- Name: `fakecheckapp`
- Description: `AI-Powered Authentication & Freshness Verification - Developed by Edmond Azbel`
- **Select PRIVATE** ← important
- Click "Create repository"
- **Why:** Private upload first means your timestamp is locked in before the world sees it

### ☑️ Step 5: Upload all files while private (3 min)
- Click "uploading an existing file"
- Drag all 13 files from your unzipped folder
- Commit message: `Initial commit - FakeCheckApp v1 by Edmond Azbel`
- Click "Commit changes"
- **Why:** Your first commit timestamp is now in GitHub's immutable history

### ☑️ Step 6: Wait 24 hours (optional but recommended)
- Don't do anything else for at least 24 hours
- This creates a clear time gap between "I uploaded" and "I published"
- **Why:** Multiple timestamps strengthen the chain of custody

### ☑️ Step 7: Make repo PUBLIC (1 min)
- Repo Settings → scroll to bottom → "Change visibility" → Make public
- Type the repo name to confirm
- **Why:** Now it's discoverable, but your earlier private timestamp is preserved

### ☑️ Step 8: Enable GitHub Pages (2 min)
- Settings → Pages → Source: main branch / root
- Save
- Wait 1-2 min for deployment
- **Why:** Your app goes live with HTTPS at `https://USERNAME.github.io/fakecheckapp/`

### ☑️ Step 9: Set repo "About" with your name (1 min)
- On repo home, click ⚙️ next to "About"
- Description: `AI-Powered Product Authentication by Edmond Azbel`
- Website: Your GitHub Pages URL
- Topics: `ai`, `pwa`, `authentication`, `edmond-azbel`, `claude`
- **Why:** Search engines index this — discoverable as your work

### ☑️ Step 10: Star your own repo (10 sec)
- Click the ⭐ Star button
- **Why:** Shows it's important to you; creates a "watched" record

## ✅ THIS WEEK — Legal Protection (60-90 min total)

### ☑️ Step 11: File CIPO Copyright Registration ($50, 15 min)
- Go to **canada.ca** → search "Register a copyright"
- Click "Apply online"
- Select "Computer program" as work type
- Upload your `index.html`
- Title: `FakeCheckApp`
- Author: `Edmond Azbel`
- Country: `Canada`
- Pay $50
- Receive registration certificate with timestamp
- **Why:** This is your strongest enforcement tool. Without it, recovering damages in court is hard. With it, statutory damages up to $20,000 per infringement become possible.

### ☑️ Step 12: Save the certificate (1 min)
- Download the CIPO certificate PDF
- Email it to yourself
- Save on multiple devices
- **Why:** This is gold in court

### ☑️ Step 13: File Trademark for "FakeCheckApp" ($330, 30 min)
- Go to **ised-isde.canada.ca** → Trademarks
- Search "FakeCheckApp" first — verify it's available (likely yes)
- File application for:
  - Class 9 (Software)
  - Class 42 (SaaS / software services)
- Cost: $330 CAD per class
- Priority date = filing date (locked in even though full registration takes 12-18 months)
- **Why:** This stops others from using your app name

- Create a new file in your repo: `INTELLECTUAL_PROPERTY.md`

## 🚨 If You Suspect Someone is Stealing Your Work

### Step 1: Document the infringement (don't confront yet)
- Screenshot their website/app/repo
- Save the URL
- Note the date/time
- Check Wayback Machine for their historical versions

### Step 2: Compare watermarks
- Did they keep ANY of your copyright notices?
- If yes: stronger case
- Did they strip ALL watermarks? That's a separate offense under Canadian Copyright Act s. 41

### Step 3: Send a cease-and-desist letter (DIY = free)
- Template available at copyrightlaws.com
- Send via certified mail (paper trail)
- Most infringers stop here

### Step 4: GitHub DMCA takedown (if they're on GitHub)
- File at github.com/contact/dmca-takedown
- GitHub usually responds within 24-48 hours
- Free

### Step 5: Consult a copyright lawyer
- If they don't stop and the value is significant
- Initial consultation often free
- Look for: "Canadian copyright lawyer" + your city

## 🎯 Reality Check

**Most likely scenario after publishing:** Nothing bad happens. Your repo gets 0-50 views. You use the app personally. You file CIPO copyright. Life continues.

**Realistic risk:** Low for v1. High-value targets are established apps with users + revenue. A brand new repo with no users is low priority for thieves.

**Highest protection comes from:** Combination of (1) timestamps + (2) CIPO copyright + (3) provisional patent + (4) being noticed (more public = more witnesses if someone tries to steal).
